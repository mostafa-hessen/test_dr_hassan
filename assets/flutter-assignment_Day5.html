نقطة البداية — تطبيقك الحالي

أنت عندك تطبيق Flutter فيه ApiService بيجيب posts من jsonplaceholder، و PostsScreen بيعرضها باستخدام setState أو FutureBuilder، و Post Model مع fromJson.
المهمة دي هتكملها بإضافة: Auth · Structured Services · Cart with Provider

المهام — خطوة بخطوة

1

التحقق من حالة تسجيل الدخول

Authentication

SplashScreen تتحقق هل المستخدم مسجل دخول وتوجهه للصفحة الصح

▼

عند فتح التطبيق، لازم تتحقق هل المستخدم مسجل دخوله أو لأ. الـ SharedPreferences بتحفظ بيانات الجلسة محلياً على الجهاز. لو مسجل → روّح لـ HomeScreen. لو مش مسجل → روّح لـ LoginScreen.

المطلوب منك:

✓

اعمل SplashScreen تظهر لثانيتين وهي بتتحقق

اعمل LoginScreen بـ email + password

لو Login نجح، احفظ بيانات الجلسة في SharedPreferences

اعمل AuthService منفصل يتعامل مع كل حاجة تخص الـ Auth

2

FutureBuilder + Structured Services

API Services

اعمل service منفصل لكل resource واستخدم FutureBuilder لعرض البيانات

▼

بدل ما يبقى كل الكود في الـ Screen، هتفصل الـ logic في Service classes منظمة. كل Service مسؤولة عن resource معين. وهتستخدم أي API مناسب لتطبيقك — سواء كانت fakestoreapi كمثال، أو أي API تاني بتاع مشروعك.

3

وظيفة الـ Logout

Auth

امسح بيانات الجلسة وروّح لـ LoginScreen

▼

الـ Logout مش بس إنك تروّح للـ LoginScreen — لازم تنظّف كل حاجة: امسح بيانات الجلسة المحفوظة، وأبعد المستخدم للـ Login.

المطلوب منك:

زر Logout في الـ AppBar أو الـ Drawer

امسح بيانات الجلسة من SharedPreferences

Navigator.pushNamedAndRemoveUntil → LoginScreen

4

Add to Cart باستخدام Provider

Provider State

CartProvider يحفظ المنتجات ويتشارك بين كل الصفحات

▼

الـ Provider بيخلي الـ Cart state متاح لكل الـ widgets في التطبيق. ProductScreen تضيف، CartScreen تعرض، AppBar يبين العدد.
